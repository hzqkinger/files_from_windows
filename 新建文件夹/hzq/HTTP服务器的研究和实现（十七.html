HTTP服务器的研究和实现（十七）

一、基本信息
　　标题：HTTP服务器的研究和实现
　　时间：2018
　　出版源：电脑编程技巧与维护
　　文件分类：文献笔记

二、研究背景
　　HTTP 协议超文本传输协议，主要用于定义 WWW 服务器传输超文本到本地浏览器， 是目前使用最为广泛的协议。HTTP 协议的出现使得互联网上不同主机之间的通信变得标准、高效， 极大地推动了互联网服务的发展。 HTTP 服务在OSI 七层网络模型中处于第 7 层应用层， 是构建于 TCP传输协议之上的应用层服务。

                                                                                           

 

三、具体内容
　1、HTTP 工作过程如下：

（1） 浏览器连接到 HTTP 服务器

（2） 发送 HTTP 请求

（3） 服务器接受处理浏览器的请求后返回 HTTP响应

（4） 释放连接 TCP 连接

（5） 浏览器解析HTML内容，并在浏览器窗口中显示。

 2、协议内容

（1） 协议部分：该URL的协议部分为 “https：” ，这代表网页使用的是HTTP协议。在" HTTP" 后面的“//” 为分隔符。

（2） 域名部分： 该 URL的域名部分为www.sogou.com,如果没有域名， 则必须是浏览器可以访问到的主机 ip。

（3） 端口部分： 域名后面是端口， 域名和端口用“:” 作分隔。 如果是默认端口 80， 则URL中可以省略端口信息。

（4） 参数部分： 从 “？” 开始到 “#” 为止之间的部分为参数部分， 又称参数部分。 本例中的参数部分为“wd=HTTP”。

                                                                              

 3、HTTP状态码

1xx： 指示信息--表示请求已成功接收， 常见的返回码如 100。

2xx： 成功--表示请求已被成功接收， 常见的返回码如 201， 200。

3xx： 重定向—表示请求被重定向到新的 URL， 常见的返回码是 301， 302，303。

4xx： 客户端错误—表示请求有语法错误或网页未找到， 常见的错误码为404。

5xx： 服务器端错误--服务器内部错误， 常见的返回码。

四、总结

　　HTTP具有简单快速的特点，当客户向服务器请求服务时，只需传送请求方法和路径。请求方法常用的有GET、HEAD、POST。每种方法规定了客户与服务器联系的类型不同。由于HTTP协议简单，使得HTTP服务器的程序规模小，因而通信速度很快。还有一个特点是支持B/S及C/S模式。

引用：陈忠菊．HTTP服务器的研究和实现．电脑编程技巧与维护．2018-08-18：133—135.
